## 🚀 Local Development

### Prerequisites

- Python 3.11+
- A [Google Gemini API key](https://aistudio.google.com/app/apikey)

### 1. Clone & Set Up Backend

```bash
git clone https://github.com/YOUR_USERNAME/StadiumMindAi.git
cd StadiumMindAi/backend

# Create virtual environment
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

### 2. Configure Environment

```bash
cp .env.example .env
# Open .env and set your GEMINI_API_KEY
```

### 3. Start the Backend

```bash
python3 run.py
# Server starts at http://127.0.0.1:8000
# API docs at   http://127.0.0.1:8000/docs
```

### 4. Build & Start the Frontend

Install Node dependencies and start the local development server:
```bash
# In the project root (where package.json is):
npm install

# Start the dev server (compiles Tailwind CSS and runs local file server)
npm run dev
```
The frontend is now served at [http://127.0.0.1:3000/code.html](http://127.0.0.1:3000/code.html) with fully compiled Tailwind CSS.

## ☁️ Deployment

### Backend → Render

> **Note:** The ChromaDB vector store is rebuilt automatically on first boot from the `/data` knowledge files. No pre-built DB is needed.

#### Option A — Render Dashboard (Recommended for first deploy)

1. Push your code to a GitHub repository
2. Go to [render.com](https://render.com) → **New** → **Web Service**
3. Connect your GitHub repo
4. Set **Root Directory** to `backend/`
5. Set **Runtime** to `Docker`
6. Under **Environment Variables**, add:
   ```
   GEMINI_API_KEY = your_actual_api_key_here
   ```
7. Click **Create Web Service**

#### Option B — render.yaml (Infrastructure as Code)

The `backend/render.yaml` is pre-configured. Render will auto-detect it when you connect your repo.

> ⚠️ **Important:** After first deploy, copy your Render service URL (e.g., `https://stadiumind-ai-backend.onrender.com`) — you'll need it for the frontend step.

---

### Frontend → Vercel

#### Option A — Vercel CLI

```bash
cd StadiumMindAi   # project root (where vercel.json lives)
npx vercel --prod
```

#### Option B — Vercel Dashboard

1. Go to [vercel.com](https://vercel.com) → **New Project**
2. Import your GitHub repo
3. Set **Root Directory** to `/` (project root)
4. Vercel will auto-detect `vercel.json`
5. Click **Deploy**

#### Connect Frontend to Backend

After deploying the backend, update the `<meta name="backend-url">` tag in `code.html`:

```html
<!-- Replace this line in code.html: -->
<meta name="backend-url" content="BACKEND_URL_PLACEHOLDER"/>

<!-- With your actual Render URL: -->
<meta name="backend-url" content="https://stadiumind-ai-backend.onrender.com"/>
```

Then redeploy the frontend:
```bash
npx vercel --prod
```

## 🔐 Environment Variables Reference

| Variable | Required | Default | Description |
|---|---|---|---|
| `GEMINI_API_KEY` | ✅ Yes | — | Google Gemini API key from AI Studio |
| `HOST` | No | `0.0.0.0` | Server bind host (set by Render automatically) |
| `PORT` | No | `8000` | Server port (set by Render automatically) |
| `CORS_ORIGINS` | No | `*` | Comma-separated allowed origins (set to your Vercel URL in production) |

## 🛠️ API Endpoints

| Method | Endpoint | Description |
|---|---|---|

| `POST` | `/api/assistant/chat` | Multilingual RAG chat |
| `POST` | `/api/operations/find-seat` | AI navigation with accessibility support |
| `POST` | `/api/operations/food-recs` | Concession recommendations |
| `POST` | `/api/operations/generate-report` | AI operations summary report |
| `GET` | `/api/analytics/incidents` | Security incident feed |
| `POST` | `/rag/reindex` | Force re-index of knowledge base documents |
